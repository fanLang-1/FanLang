## ADR-001: 定位转变

**日期**: 2026-07
**状态**: ✅ 已接受

**背景**: 初始商业计划将凡语定位为面向普通用户的 AI 工具，对标豆包/Kimi。

**决策**: 定位转变为"为开源 AI 项目赋能的交互层"。

**理由**:
1. 豆包/DeepSeek/Kimi 免费且模型更强，C端没赢面
2. 安装 Ollama+Python 本身对普通用户是门槛
3. 开源项目开发者才是真正需要"标准化AI交互组件"的用户
4. 25 符号=能力菜单，OODA=交互引擎，术语表=本地化SDK——三个组件恰好构成一套"AI设计的Ant Design"

**替代方案**:
- C端工具（已否决：竞品碾压）
- 政企私有部署（保留为远期方向）
- 学术研究工具（保留为远期方向）

**后果**: 产品形态从"一个App"变成"三个pip包+文档体系"。

## ADR-002: 三包独立

**日期**: 2026-07
**状态**: ✅ 已接受

**背景**: 所有功能放在一个包里 (fanlang)，导致 Streamlit 和 LangChain 用户被强制安装不需要的依赖。

**决策**: 拆为三个独立 pip 包，放在一个 monorepo。

**理由**:
1. LangChain 用户不需要 Streamlit，反之亦然
2. 核心包 (fanlang) 依赖最轻 (仅 requests)，可以嵌入任何 Python 项目
3. 独立包更容易被各自生态的社区发现和引用
4. monorepo 保证同步发布和统一版本号

**替代方案**:
- 单包+extras_require（已否决：不够解耦，install 体验差）
- 完全独立仓库（已否决：同步困难）

**包依赖关系**:
```
fanlang (核心，零外部强依赖)
├── fanlang-streamlit (依赖 fanlang + streamlit)
└── fanlang-langchain (依赖 fanlang + langchain)
```

## ADR-003: 七角色

**日期**: 2026-07
**状态**: ✅ 已接受

**背景**: 研讨厅需要预制角色，让用户一行代码启动。

**决策**: 7 个工厂角色——王写手/李编辑/张校对/刘顾问/陈裁判/赵创意/孙批判。

**理由**:
1. 覆盖三类组织形态的最小需求（Pipeline需要3个，Panel需要3+1，Adversarial需要2+1）
2. 每个角色绑定不同凡语符号，确保认知多样性
3. 命名采用中文姓氏+职能，有亲和力
4. 7个不多不少——再多用户记不住，再少不够用

**角色×符号映射**:
```
王写手 → 写 (产出者)
李编辑 → 改 (润色者)
张校对 → 查 (校验者)
刘顾问 → 理 (分析者)
陈裁判 → 评 (裁决者)
赵创意 → 想 (创意者)
孙批判 → 解 (批判者)
```

**替代方案**:
- 12 个角色（已否决：过多，用户迷失）
- 4 个角色（已否决：不够覆盖 Adversarial）
- 不给预制角色（已否决：门槛太高）

## ADR-004: 归档策略

**日期**: 2026-07
**状态**: ✅ 已接受

**背景**: 两个原始项目 (zhongwen-ai-lang / ebook-rag) 与新 monorepo 结构冲突。

**决策**: 原始代码归档到 `legacy/` 目录，保留完整功能但不作为主产品。

**理由**:
1. 保留历史——两个项目是凡语的 genesis
2. start.bat 等仍可独立运行，不影响老用户
3. 新代码在 fanlang/ 下重构，不受 legacy 污染
4. legacy 是活文档——展示了凡语从何而来

## ADR-005: 文档体系

**日期**: 2026-07
**状态**: ✅ 已接受

**背景**: AI 会话之间没有上下文持久化，每次需要重新解释项目状态。

**决策**: 建立 guide/ 目录，作为 AI 的"持久化记忆体"。凡语符号作为章节标记。

**理由**:
1. 文档天然跨会话——AI读取即恢复上下文
2. 凡语符号做章节标记 → 文档本身展示产品能力
3. 用户不需要记项目状态，改文档即可
4. 拆分六份文档 (CONTEXT/SCENARIOS/DECISIONS/PATTERNS/ROADMAP/GOVERN)，比一个巨大 README 更容易维护

**文档流转规则**: 见 GOVERN.md

**替代方案**:
- 单一大 README（已否决：太长，没人读）
- 数据库存状态（已否决：过度设计）
- Git commit message（已否决：碎片化，不可检索）

## ADR-006: 命名

**日期**: 2026-07
**状态**: ✅ 已接受

**背景**: 商业计划中临时命名为"语墨"，后改为"凡语"。

**理由**:
1. "凡语" = 凡人的语言——不需要学prompt，用平凡的字
2. "凡" 通 "繁"——简中见繁，25单字背后是完整体系
3. 与 GitCode 仓库名 `FanLang` 一致
4. 比"语墨"更有文化叙事张力
