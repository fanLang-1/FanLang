# 凡语 · 文档治理规则

> 这份文档定义 guide/ 体系如何创建、更新、流转。
> AI 读取本文件后，知道怎么维护整个文档体系。

---

## 【定】文档结构

```
guide/
├── CONTEXT.md      ← 入口。AI 每次会话第一读。
├── SCENARIOS.md    ← 场景映射。AI 执行任务时索引。
├── DECISIONS.md    ← 架构决策。记录为什么这么做。
├── PATTERNS.md     ← 经验模式。可复用的代码和教训。
├── ROADMAP.md      ← 路线图。接下来做什么。
└── GOVERN.md       ← 本文件。文档怎么维护。
```

---

## 【阅】读取规则（给 AI）

1. **每次新会话开始**：必须先读 `CONTEXT.md`。
2. **执行任务前**：根据任务类型索引 `SCENARIOS.md`。
3. **做设计决策前**：查 `DECISIONS.md` 避免推翻已有决策。
4. **写代码前**：查 `PATTERNS.md` 复用现有模式。
5. **不确定优先级时**：查 `ROADMAP.md`。
6. **需要维护文档时**：查本文件。

---

## 【改】更新规则（给 AI）

| 发生了什么事 | 更新哪个文件 |
|:--|:--|
| 新功能完成 | CONTEXT.md【改】区域 + ROADMAP.md 勾选 |
| 做了设计决策 | DECISIONS.md 新增 ADR |
| 学到新经验/模式 | PATTERNS.md 新增条目 |
| 重大架构变更 | CONTEXT.md【查】区域 + DECISIONS.md |
| 版本发布 | CONTEXT.md 版本号 + ROADMAP.md 状态 |
| 发现新场景 | SCENARIOS.md 新增场景行 |

## 【记】格式规则

### 符号作为章节标记

所有 guide/ 文档使用凡语符号作为二级标题前缀：

```
## 【总】 = 总结/综述
## 【查】 = 查询/现状
## 【改】 = 变更记录
## 【想】 = 设计思路/未决
## 【定】 = 决策/路线
## 【写】 = 详细描述
## 【理】 = 分类/表格
## 【比】 = 对比/分析
## 【编】 = 代码示例
## 【记】 = 规则/备忘录
## 【解】 = 解释/原理
## 【阅】 = 文档导航/阅读指引
```

### 不允许的格式

- 复杂的 Markdown 嵌套 (3 层以上)
- 内嵌 HTML（保持纯 Markdown 可读性）
- 无意义的表情符号（凡语符号本身够用）

## 【转】流转规则

### 会话开始

```
1. AI 读取 guide/CONTEXT.md
2. 理解项目状态和当前任务
3. 根据 SCENARIOS.md 选择执行策略
4. 执行任务
```

### 会话进行中

```
1. AI 做决策 → 写入 DECISIONS.md
2. AI 发现模式 → 写入 PATTERNS.md
3. AI 完成功能 → 更新 CONTEXT.md【改】区域
```

### 会话结束

```
1. AI 更新 CONTEXT.md【查】区域（最新状态）
2. AI 更新 ROADMAP.md（完成项→勾选）
3. AI 提交所有文档变更
4. 推送
```

---

## 【查】文档健康检查

| 检查项 | 频率 |
|:--|:--|
| CONTEXT.md 状态是否准确 | 每次结束前 |
| DECISIONS.md ADR 总数 | 每个重大决策后 |
| PATTERNS.md 是否有重复模式 | 每月 |
| ROADMAP.md 时间线是否现实 | 每次发布后 |
| 文档总字数 (目标: 2-3万字) | 每季度 |

---

## 【空】文档创建模板

新建文档时，必须：

1. 文件名大写英文（如 `XXX.md`）
2. 开头一句话概括（`> 一句话`）
3. 使用凡语符号做二级标题
4. 在 CONTEXT.md【阅】区域添加链接
5. 在 GOV.md 本文件【阅】区域添加读取规则
